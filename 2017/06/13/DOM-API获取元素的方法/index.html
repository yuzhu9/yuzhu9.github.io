DOM-API获取元素的方法

发表于 2017-06-13 分类于 笔记

文档对象模型（DOM）

文档对象模型 (DOM) 是 HTML 和 XML 文档的编程接口。它给文档（结构树）提供了一个结构化的表述并且定义了一种方式—程序可以对结构树进行访问，以改变文档的结构，样式和内容。 DOM 提供了一种表述形式— 将文档作为一个结构化的节点组以及包含属性和方法的对象。从本质上说，它将 web 页面和脚本或编程语言连接起来了。

document.getElementById()返回一个匹配特定ID的元素节点。

element是一个 Element 对象。如果当前文档中拥有特定ID的元素不存在则返回null。
id是大小写敏感的字符串，代表了所要查找的元素的唯一ID。

document.getElementsByTagName()返回一个包括所有给定标签名称的元素的HTML集合。

elements是一个由发现的元素出现在树中的顺序构成的动态的HTML集合(最新的 W3C 规范 说明这些元素是 HTMLCollection（HTML集合）； 然而, 这个方法在WebKit内核的浏览器中返回一个 NodeList 。)
name是一个代表元素的名称的字符串。特殊字符 “*” 代表了所有元素。

element.getElementsByTagName()方法返回一个动态的包含所有指定标签名的元素的HTML集合HTMLCollection。

与’document.getElementsByTagName()’类似，只是它的搜索范围被限定在指定元素的后代。
指定的元素的子树会被搜索，不包括元素自己。返回的列表是动态的，这意味着它会随着DOM树的变化自动更新自身。如果没有找到元素，HTMLCollection为空
如果是HTML文档中的某个元素调用了getElementsByTagName函数， 运行前会将参数转为小写字母形式。

getElementsByClassName()返回一个类似数组的对象，包含了所有指定 class 名称的子元素。

elements 是查找到的所有元素的集合HTMLCollection。
names 是一个字符串，表示用于匹配的 class 名称列表; class 名称通过空格分隔
getElementsByClassName 可以在任意的元素上调用，不仅仅是 document。 调用这个方法的元素将作为本次查找的根元素.

getElementsByName()根据给定的name 返回一个在 (X)HTML document的节点列表集合。

elements 是一个 NodeList 集合。
name 是元素的 name 属性的值。
name属性仅适用于（X)HTML文档。该方法返回一个动态的NodeList集合，包含所有具有指定的name属性值的元素。
在IE或Opera中，getElementsByName()方法也将返回具有指定值的id属性的元素。因此不要将name和ID值设为相同的字符串。

document.querySelector()返回文档中匹配指定的选择器组的第一个元素节点

selectors 是一个字符串，包含一个或是多个 CSS 选择器 ，多个则以逗号分隔。
如果没有找到匹配元素，则返回 null，如果找到多个匹配元素，则返回第一个匹配到的元素。
传递给 querySelector 的字符串参数必须符合 CSS 语法。
CSS 伪类不会返回任何元素。

element.querySelector()返回调用元素的后代中与指定的选择器组匹配的第一个元素

返回的是baseElement后代元素中第一个与指定的选择器组匹配的元素。
如果没有找到匹配元素，则返回 null

document.querySelectorAll()返回与指定的选择器组匹配的文档中的元素列表。返回的对象是 NodeList

selectors 是一个由逗号连接的包含一个或多个CSS选择器的字符串。
传递到querySelectorAll()的字符串参数必须遵循CSS语法。

element.querySelectorAll()返回一个non-live的NodeList, 它包含所有元素的非活动节点

node.parentNode返回指定的节点在DOM树中的父节点

每个节点都有一个parentNode属性，它表示元素的父节点。
一个元素节点的父节点可能是一个元素(Element )节点,也可能是一个文档(Document )节点,或者是个文档碎片(DocumentFragment)节点.

node.parentElement返回当前节点的父元素节点

返回元素的父元素节点，与parentNode的区别在于，其父节点必须是一个Element
如果该元素没有父节点,或者父节点不是一个元素节点.则返回null

element.previousElementSibling返回当前元素在其父元素的子元素节点中的前一个元素节点

如果该元素已经是第一个元素节点,则返回null
该属性是只读的

element.nextElementSibling返回当前元素在其父元素的子元素节点中的后一个元素节点

如果该元素已经是最后一个元素节点,则返回null
该属性是只读的

node.children返回一个Node的子元素的动态HTMLCollection

children 属性为只读属性，对象类型为HTMLCollection

node.firstElementChild返回对象的第一个子元素节点

该属性是只读的
如果没有子元素，则为null

node.lastElementChild返回对象的最后一个子元素节点

该属性是只读的
如果没有子元素，则为null

document.documentElement会返回文档对象（document）的根元素，例如HTML文档的根元素是元素，该属性是只读属性

以上只列举了一部分方法，更多内容需查看MDN
