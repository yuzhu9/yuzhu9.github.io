语法

1.ECMAscript中的一切（变量、函数名和操作符）区分大小写
事件作为HTML的属性时，可以不区分大小写，但事件所对应的内容要区分大小写

2.标识符，是指变量、函数名、属性的名字，或者函数的参数
标识符的第一个字符必须是一个字母(可以是世界上任何一种语言，包括汉字)、下划线（_）或美元符号（$）
其他字符可以是字母、下划线、美元符或者数字

3.注释包括单行注释和多行注释

5.语句以一个分号结尾，如果省略分号，则由解析器确定语句的结尾
解析器判断语句结尾的过程：
（1）首先判断后一行代码能否与前一行一起执行，如果可以，合并到一起执行
（2）如果不能执行，就分别执行两行代码
（3）如果分别执行仍然不能正确执行，则报错
因此建议在每一条语句前面加分号，以分号开头，避免其他代码未以分号结束而引起错误
return break continue这三个关键字不会与下一行代码合并，因此这三个关键字后面的代码需要紧跟在这三个关键字后而不能换行

6.关键字和保留字
关键字：一组具有特定用途的关键字，可用于表示控制语句的开始或结束，或者用于执行特定操作等，这些关键字不能被用作标识符。
保留字：暂时没有特定的用途，但是有可能在将来被用作关键字。（Java的关键字都是JavaScript的保留字）
命名标识符时为了避开关键字和保留字冲突，可才用驼峰式命名，在实际开发中，使用拼音命名也很常见

7.变量
ECMAScript的变量是松散型的。每个变量仅仅是一个用于保存值的占位符而已。
定义变量时，变量并没有定义类型。在使用typeof方法时才定义变量的类型

8.简单数据类型和复杂数据类型
(1)5种简单数据类型：Undefined Null Boolean Number String
(2)1种复杂数据类型：Object

在JavaScript中，所有基础类型都是不可变的，不可变的数据进行比较，是值进行比较
任何两个独立的对象都不相等，如果要比较对象，是对对象的引用进行比较，比较的是引用关系
理论上，只有对象有方法，其他5种基本类型都没有方法

当调用字符串的方法的时候，解析器会把字符串临时转为对象，但调用结束结果不保存
数值、字符串、布尔这三种基本数据类型在调用方法的时候，是转换为对象来调用的。

typeof是操作符，typeof所有的返回值都是字符串。

undefined和null的共同点：
(1)都只有一个值
(2)当这两个值参与判断，都返回false
(3)这两个值都没有方法

不同点：
(1)null是关键字，已初始化；undefined不是关键字，未初始化
(2)null是特殊对象，是空对象，undefined是 window 对象的属性，这个属性对应的值是未定义
(3)typeof null的值是 ‘object’，typeof undefined的值是 ‘undefined’
(4)null转换为数值类型是0，undefined是 NaN
