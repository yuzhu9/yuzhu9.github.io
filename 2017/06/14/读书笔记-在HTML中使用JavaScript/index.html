读书笔记-在HTML中使用JavaScript

发表于 2017-06-14 分类于 笔记

1.使用<script>元素是向HTML页面中插入JavaScript的主要方法。

2.<script>元素的属性：

  • async 异步下载，不推荐使用
  • defer 延迟执行，不推荐使用
  • charset 一般不用，出现乱码可能使用，但概率不高
  • src 表示包含要执行代码的外部文件，是最常用最重要的属性。
  • type 表示编写代码使用的脚本语言的内容类型（也称为MIME类型）。通常使用的是”text/javascript”,是默认值，可以省略。实际上，服务器在传送JavaScript文件时使用的MIME类型通常是application/x-javascript，但在type中设置这个值却可能导致脚本被忽略。

3.包含在<script>元素内部的JavaScript代码将被从上至下依次解释
解释的意思是预处理和执行。
（1）JS是预处理后执行
（2）预处理+执行是阻断式操作
（3）预处理+执行就是解释
也就是说JS代码从上至下按照代码块来预处理和执行

4.在使用<script>嵌入JavaScript代码时，不要在代码中的任何地方出现</script>字符串，如果有需通过转义符\来解决。

5.<script>标签内既引用外部标签，内部也写了JS代码时，浏览器执行时一般会执行外部文件，而忽略标签内部的JS代码。

6.外部JavaScript文件一般需要带有.js扩展名。但这个扩展名不是必须的，如果不适用.js扩展名，需要确保服务器能返回正确的MIME类型。

7.<script>元素的scr属性还可以包含来自外部域的JavaScript文件——跨域

8.实际开发中，<script>标签通常放在页面的后面，因为JavaScript解析是阻断式的（包括下载），放在前面会影响页面的正常加载。
除非遇到必须把JavaScript文件文件放在前面的情况，否则不要放在前面（例如一些与CSS相关的JS，或者某些框架,JavaScript需要在页面渲染之前执行）

9.嵌入代码与外部文件的优缺点
外部文件：可维护性、可缓存、适应未来
内部文件：相对外部文件的优点为性能提升，尤其是在手机等移动设备上，减少了连接数，提升效果明显

10.文档模型：混杂模式（quirks mode）和标准模式（standards mode）。
虽然这两种模式主要影响CSS内容的呈现，但在某些情况下，也会影响到JavaScript的解释执行
