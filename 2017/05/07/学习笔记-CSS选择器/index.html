学习笔记-CSS选择器

发表于 2017-05-07 分类于 笔记

基本选择器

  • 1.通配选择器* 匹配任何元素
  • 2.元素选择器element匹配所有使用该标签的元素
  • 3.ID选择器#id
  • 4.类选择器.class匹配所有使用该类名的元素

    组合选择器

  • 1.多元素选择器A,E同时匹配所有A元素或E元素
  • 2.后代选择器A E匹配所有属于A元素后代的E元素
  • 3.子元素选择器A>E匹配所有A元素的子元素E
  • 4.相邻兄弟选择器B+E匹配所有紧随B元素之后的同级元素E
  • 5.通用兄弟选择器E ~ FE和F是同辈元素，具有相同的父元素，并且F元素在E元素之后，E ~ F将选中E元素后面的所有F元素

    伪类选择器

1.动态伪类选择器

  • E:link 选择定义了超链接但链接还未被访问过的元素
  • E:visited 选择定义了超链接并且链接已经被访问过的元素
  • E:active 选择匹配的E元素，且元素被激活，常用在锚点和按钮上
  • E:hover 选择鼠标停留的匹配的E元素
  • E:focus 选择匹配的E元素，且元素获得焦点

2.目标伪类选择器

  • :target 戳我查看

3.语言伪类选择器

  • :lang 用来选择制定了lang属性的元素

4.状态伪类选择器

  • E:checked 匹配表单中被选中的单选按钮或复选按钮
  • E:enabled 匹配所有启用的表单元素
  • E:disabled 匹配所有禁用的表单元素

5.结构伪类选择器

  • :root 选择文档的根元素(HTML文档的根元素就是html元素)

  • E:nth-child() 选择其父元素的一个或多个特定的子元素,括号内的值可以是数值（从1开始），也可以是包含n的表达式，也可以是odd（奇数）、even（偶数）

  • E:nth-last-child() 使用方法与:nth-child()相同，不同的是选择元素的顺序从父元素的最后一个子元素开始算

  • E:nth-of-type()只计算父元素中指定的某种类型的子元素，当某个元素的子元素类型不只是一种时，使用nth-of-type来选择会比较有用

  • E:nth-last-of-type()用法与E:nth-of-type()相同，不同的是它是从父元素的最后一个子元素开始算起

  • E:first-child 匹配父元素的第一个子元素，与E:nth-child(1)效果一样

  • E:last-child 匹配父元素的最后一个子元素

  • E:first-of-type类似于:first-child，不同的就是指定了元素的类型

  • E:last-of-type类似于:last-child，不同的就是指定了元素的类型

  • E:only-child匹配的元素E是其父元素的唯一子元素，也就是说匹配元素的父元素只有一个子元素

  • E:only-of-type用来选择一个元素，他的类型在他父元素的所有子元素中是唯一的。也就是说，一个父元素有很多子元素，而其中只有一个子元素的类型是唯一的，那么就可以使用:only-of-type来选取这个元素

  • E:empty用来选择没有任何子元素的元素，文本节点也被看作子元素

6.否定伪类选择器

  • E:not(F)可以用来选取所有除了F外的所有元素。

  • 2.::first-line伪元素只能用于块级元素，下面的属性可应用于 first-line 伪元素

  • font properties

  • 4.::after伪元素可以在元素的内容后面插入新内容

    属性选择器

  • 1.E[attr]

  • 2.E[attr="val"]

  • 3.E[attr~="val"]当元素的某个属性具有多个用空格隔开的属性值，此时使用E[attr~=val]只要attr属性多个属性值中有一个于val匹配元素就会被选中。

  • 4.E[attr|="val"]用来选择具有属性attr且属性的值为val或以val-开头的元素(其中-是不可或缺的)。

  • 5.E[attr^="val"]用来选择属性attr的属性值是以val开头的所有元素

  • 6.E[attr$="val"]这个选择器刚好跟E[attr^=val]相反，用来选择具有attr属性且属性值以val结尾的元素

  • 7.E[attr*="val"]这个属性选择器使用了通配符，用来选择具有属性attr并且只要属性值中包含val字符串的元素。也就是说只要所选属性中有val字符串，不管是不是多个用空格分隔的属性值，都将被选中
