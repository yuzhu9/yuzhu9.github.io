浅析URL

发表于 2021-01-05 分类于 笔记

URL（Uniform Resource Locator）统一资源定位符，它是由HTML的发明者蒂姆·伯纳斯-李发明的，用来作为万维网的地址，URL是由它包括以下几部分：

  1. 协议：HTTP协议、HTTPS协议

    HTTP协议的笔记

  2. 域名或IP

    • IP（Internet Protocol）互联网协议，主要约定了两件事：一是如何定位一台设备，二是如何封装数据报文，以跟其他设备交流。IP协议规定的网络地址就是IP地址，就是用来标识发送或接受数据的设备的一串数字

    几个特殊的IP地址分别是：

    • ping命令可以用来测试本机与目标主机是否连通，连通的速度、稳定性

    • 域名就是给IP起的别称，因为IP地址是一串数字，不方便记忆使用，例如百度的域名是baidu.com

      • 一个域名可以对应不同的IP，这称为均衡负载，防止一台机器扛不住，一般是大公司使用

      • 一个IP可以对应不同的域名，这称为共享主机，节约money

      • 域名是分层级的，例如www.baidu.com

    • DNS（Domain Name System)域名系统，域名和IP是通过DNS对应起来的，DNS服务由DNS服务器提供，更多内容阮一峰-DNS原理入门

    • nslookup是一个能够查询互联网域名服务器信息的程序，使用nslookup命令加上所要查询的域名就可以查询到对应的IP地址，更多nslookup功能nslookup命令使用详解

  3. 端口号

    一台机器可以提供很多不同的服务，对应不同的端口，一共有65535个端口

    端口的规则是：

    • 0到1023号端口是留给系统使用的
    • 拥有了管理员权限，才能使用这1024个端口
    • 除了这1024个端口之外的其他端口可以给普通用户使用
    • 一个端口如果被占用，就只能换一个端口

    在浏览器中，如果端口号是使用的默认端口号，可以省略不写，浏览器会自动补齐

  4. 路径

    通过设置不同的路径可以请求同一个服务器上不同的页面

  5. 查询参数

    通过设置不同的查询参数可以请求同一个页面的不同内容

  6. 锚点

    通过设置不同的锚点可以定位到同一个页面同一个内容上的不同位置

    锚点是不会传给服务器的，它是由浏览器来处理的
