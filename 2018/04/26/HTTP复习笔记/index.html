1.HTTP是什么

HTTP即超文本传输协议（HyperText Transfer Protocol），是万维网数据通信的基础。它的作用是指导浏览器和服务器之间如何进行沟通。

举个栗子：当我们在地址栏输入网址（如：www.baidu.com)，回车之后就可以看到对应的网页。这个过程简单来讲，就是浏览器发出请求，服务器对该请求进行响应并返回内容。HTTP在其中的作用就是告诉浏览器如何发送请求，服务器如何响应请求。

2. HTTP请求与响应

HTTP请求包括四部份：

  • 1.请求行：请求动词 路径 协议/版本号

  • 2.请求头部：键值对形式的头部字段（如 Host: www.baidu.com ）

  • 3.空行（回车）

  • 4.请求数据：请求数据可以为空

  • 1.状态行：协议/版本号 状态码 状态名词

  • 3.空行（回车）

  • 4.响应数据：响应内容的格式由第二部分响应头部的Content-Type标注

4.curl命令的简单使用

  • curl是一种命令行工作，作用是发出网络请求，然后得到和提取数据，显示在“标准输出”上。它支持多种协议。

  • curl www.baidu.com或curl -- www.baidu.com会显示网页的源码

  • curl -L自动跳转新的网址

  • curl -o baidu.html www.baidu.com会将网页内容下载到文件baidu.html中

  • curl -v 显示通信的整个过程，也就是会显示请求和响应，如果不加-v则只显示响应不显示请求

  • curl -H "Key: value"添加请求头

  • curl -s不显示下载进度

  • curl -X POST默认的请求动词是GET ，使用-X可以更换为其它动词，目前有7个，分别是GET/POST/PUT/PATCH/OPTION/DELETE/HEAD

  • curl -d xxx传送数据

  • 更多待补充

    最后举个栗子：例如在命令行输入命令curl -X POST -d testdata -s -v -H "Key: value" -- www.baidu.com就会得到如下图结果

    以POST方式（-X POST）向www.baidu.com（-- www.baidu.com）发送请求，发送的数据内容为testdata(-d testdata)，发送的请求添加了请求头部Key: value(-H "Key: value")，在标准输出界面显示了请求和响应的内容（-v）而未显示下载进度(-s)
